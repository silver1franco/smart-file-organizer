# Contributing

Thanks for your interest in contributing!

## Reporting Bugs

Open an issue with:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Your OS and Python version

## Submitting Changes

1. Fork the repo
2. Create a branch (`git checkout -b fix-something`)
3. Make your changes
4. Test manually with `--dry-run` on a test directory
5. Submit a pull request

## Code Style

- Keep it simple and readable
- Use type hints for function signatures
- Follow existing patterns in the codebase
- No external dependencies (stdlib only)

## Ideas for Contributions

- Additional file categories
- New organization strategies (by size, by project, etc.)
- Performance improvements for large directories
- Better duplicate detection (first-chunk comparison before full hash)

## Questions?

Open an issue and ask. Happy to help.
